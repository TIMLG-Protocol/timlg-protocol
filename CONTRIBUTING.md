# Contributing

Thanks for your interest in TIMLG.

This repository is the **public documentation** site. Contributions are welcome for:

- typos and clarity improvements
- diagrams and explanations
- formatting and site UX

## Guidelines

- Do not add anything sensitive (keys, signer paths, privileged configs).
- Keep content aligned with the current MVP behavior described in Protocol pages.
- For larger changes, open an issue first.

## Workflow (simple)

1. Fork the repo
2. Create a branch: `docs/<topic>` or `chore/<topic>`
3. Open a pull request
